Functions added or changed in js/main.js
- [restoreTemplateAwareTextForBulkExport](js/main.js#L617)
	- Added code that finds nodes with `data-template-text` and restores the original template sentence before bulk export snapshot capture.
	- Added a special case for `custom-heading` so its text is restored with `textContent` instead of inner HTML.
- [buildTemplateAwareBinding](js/main.js#L1266)
	- Added logic to detect sentence-style bindings from live canvas text.
	- Added fallback template reconstruction when the node already contains resolved value text instead of explicit `{name}` markers.
	- Added `syntheticPath` generation so the frontend can separate template-aware values from normal JSON fields.
- [exportDesignAndSend](js/main.js#L1488)
	- Added `normalizeTemplateWhitespace` to convert `&nbsp;`, `&#160;`, and `\u00a0` into spaces.
	- Added `ensureZipLibrary` so a ZIP library with `loadAsync` can be loaded at runtime for post-processing the download.
	- Added `filterTemplateAwareZipEntries` to inspect the returned ZIP and remove placeholder-style helper entries before the file is downloaded.
	- Added `downloadBlob` handling so the filtered archive is downloaded instead of the raw backend ZIP when template-aware entries exist.
	- Added guards that skip stale helper uploads whose names contain `__i_designer_template_values`.
	- Added JSON enrichment so the resolved sentence value is written into `__i_designer_template_values` for the backend render step.
- [normalizeTemplateWhitespace](js/main.js#L1531)
	- Added replacements for HTML entities and unicode non-breaking spaces.
- [ensureZipLibrary](js/main.js#L1557)
	- Added lazy script injection for JSZip 3.10.1 when the loaded version does not support `loadAsync`.
- [filterTemplateAwareZipEntries](js/main.js#L1586)
	- Added ZIP inspection with `loadAsync`.
	- Added a filter that keeps only entries whose names match the template-aware helper output path.
	- Added regeneration of a cleaned ZIP blob for download.

Other js/main.js changes
- Template-aware binding detection was added in the bulk modal flow so resolved sentence bindings are handled separately from normal JSON-path bindings.
- Bulk upload handling now ignores stale synthetic helper file names containing `__i_designer_template_values`.
- The response download path now filters the ZIP so the placeholder English file is not downloaded when the resolved template output exists.
